素数分布-RSA相关

Posted on 2018-03-05

在RSA的的算法中间，有一个非常核心的问题，就是要找两个足够大的素数，利用这两个素数的乘积很难用计算机在短时间完成分解的性质来设计的。 但是这边就有一个非常现实的问题，第一，这两个足够大的素数怎么找，第二，素数够不够多，如果素数数量是非常有限的，随着数越来越大，素数的个数增长也同时变得很慢，那是不是就可以利用列表的方式，然后用\(n^2\)的复杂度来完成暴力破解

素数的分布

这边首先要提一个Prime number theroem。这个理论直接揭示了素数分布的规律，简单说素数的分布符合以下的分布

Wiki上面对这个分布，有一个形象的说法，就是说对于一个小于\( 10^n \)的数来说，它是素数的概率是一个小于\( 10^{2n} \)的数的两倍

然后根据这个分布，我们可以得到大概的素数个数的估计，已经大概的素数间隔的估计

范围 素数个数 平均间隔估计
\(10^5\) \(9592\) \(10.425\)
\(10^{10}\) \(455 052 511\) \(21.975\)
\(10^{20}\) \(2 220 819 602 560 918 840\) \(45.028\)

因为现在key的size大概是 \(1024\) bit - \(4096\) bit, 这就意味着我们使用的素数大概是 \(512\) bit - \(2048\) bit。 \(2^{512}\) 大于 \(10^{150}\)。 利用上面的分布，大概我们可以估计出在这区间中大概有\(2.8\times10^{147}\)个素数，这个数量是非常巨大的，几乎不可能用暴力搜索的办法来分解两个大素数的乘积

如何寻找这两个素数

上面已经提到素数的平均间隔其实是有限的，所以我们就有可能采取下面的算法

  1. 随机产生一个\(512\) bit的奇数 \(p\)
  2. 判断这个数是质数与否，这边大概需要判断\( \frac{log(p)}{2} \sim 177 \)次
  3. 如果不是， 加 \(2\)， 继续判断
